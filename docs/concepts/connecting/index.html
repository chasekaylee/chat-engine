Connect to PubNub

This section outlines how to set up and initialize a PubNub instance, how to connect with a user, set metadata for that user, disconnect, and reconnect.

Setup

Start by including the PubNub SDK in your code before you initialize the client.

JavaScript
Swift
Java
<script src="https://cdn.pubnub.com/sdk/javascript/pubnub.4.23.0.js"></script>
/**
 * Create Podfile in project root directory with following
 * content:
 */
platform :ios, '9.0'
use_frameworks!

pod 'PubNub', '~> 4.0'

target 'TargetName' do
end
 
 
/**
 * Make sure to replace 'TargetName' with your project target name
 * before running following command with Terminal from project
 * root directory:
 */
pod install

 
/**
 * CocoaPods will create .xcworkspace file in project root
 * directory which you should use from now on.
 */
implementation 'com.pubnub:pubnub-gson:4.22.0-beta'

Initializing PubNub

You need to include the publishKey and subscribeKey when you initialize PubNub from your application. You can access your keys from the Admin Dashboard.

If a client will only receive messages, then it only needs the subscribeKey. If a client will be sending messages as well, it will need both the publishKey and subscribeKey.

JavaScript
Swift
Java
const pubnub = new PubNub({
  subscribeKey,
  publishKey,
});
let configuration = PNConfiguration(publishKey: publishKey,
                                    subscribeKey: subscribeKey)
configuration.stripMobilePayload = false
let pubnub = PubNub.clientWithConfiguration(configuration)
PNConfiguration pnConfiguration = new PNConfiguration();
pnConfiguration.setSubscribeKey(SUB_KEY);
pnConfiguration.setPublishKey(PUB_KEY);

PubNub pubNub = new PubNub(pnConfiguration);

Setting a UUID for each user

You also need to specify a UUID when initialize PubNub. The UUID should be unique for each user and should remain unchanged for the lifetime of the user. If your application doesn't set a UUID, the SDK will automatically generate a random UUID on every login.

Often, UUID is the primary identifier for the user in your database but it can be any string. For example, if your login method is a social login, such as Twitter OAuth, you can assign the username as UUID.

Caution! Don't use email addresses as user IDs. Users may change their email address, and it also unnecessarily exposes private information.

JavaScript
Swift
Java
const pubnub = new PubNub({
  subscribeKey,
  publishKey,
  uuid,
});
// TODO: split this sample into initialization and setting UUID

let configuration = PNConfiguration(publishKey: publishKey,
                                    subscribeKey: subscribeKey)
configuration.stripMobilePayload = false
configuration.uuid = uuid

let pubnub = PubNub.clientWithConfiguration(configuration)
PNConfiguration pnConfiguration = new PNConfiguration();
pnConfiguration.setSubscribeKey(SUB_KEY);
pnConfiguration.setPublishKey(PUB_KEY);
pnConfiguration.setUuid(uuid);

PubNub pubNub = new PubNub(pnConfiguration);

Setting state for a user

Note: Enable Presence from the Admin Dashboard.

Use the setState() method to set extra information for a user on a channel. PubNub stores the state for the time the user is online. When you set or update the state, a state-change event is fired to other users on the channel.

The state can be an object of key/value pairs with any arbitrary data, as long as the data is of type int, float, or string.

JavaScript
Swift
Java
pubnub.setState({
  state: {
    mood: 'grumpy',
  },
  channels: ['room-1'],
}, (status, response) => {
  // handle state setting response
});
pubnub.state().set()
  .state([ "mood": "grumpy" ])
  .channels(["room-1"])
  .performWithCompletion({ status in
    // handle state setting response
  })
JsonObject state = new JsonObject();
state.addProperty("mood", "grumpy");

pubNub.setPresenceState()
        .state(state)
        .channels(Collections.singletonList("room-1"))
        .async(new PNCallback<PNSetStateResult>() {
            @Override
            public void onResponse(PNSetStateResult result, PNStatus status) {
                if (!status.isError()) {
                    // handle state setting response
                }
            }
        });

Call the getstate() method to get a user's state info:

JavaScript
Swift
Java
pubnub.getState({
  channels: ['room-1'],
}, (status, response) => {
  // handle state getting response
});
pubnub.state().audit()
  .channels(["room-1"])
  .performWithCompletion({ (result, status) in
    // handle state getting response
})
pubNub.getPresenceState()
        .channels(Collections.singletonList("room-1"))
        .async(new PNCallback<PNGetStateResult>() {
            @Override
            public void onResponse(PNGetStateResult result, PNStatus status) {
                if (!status.isError()) {
                    // handle state setting response
                }
            }
        });

Reconnecting to PubNub

On most platforms (including Swift and Android), the SDK automatically reconnects if reconnection policy flags are enabled when the PubNub instance is initialized.

The JavaScript SDK relies on the browser networking events for reconnection tries, it does not ping on regular intervals in order to conserve resources. It generates browser networking events if autoNetworkDetection flag is enabled, and catches up missed messages from the queue if restore flag is enabled you initialize PubNub.

  • When the network vanishes, the SDK sends PNNetworkDownCategory
  • When the network comes back, the SDK sends PNNetworkUpCategory

To re-establish a connection with PubNub, call pubnub.reconnect() when the PNNetworkUpCategory event is received.

JavaScript
Swift
Java
const pubnub = new PubNub({
  subscribeKey,
  publishKey,
  // enable for non-browser environment automatic reconnection
  autoNetworkDetection: true,
  // enable catchup on missed messages
  restore: true,
});  
/**
 * If connection availability check will be done in other way,
 * then use this  function to reconnect to PubNub.
 */
pubnub.reconnect();
let configuration = PNConfiguration(publishKey: publishKey,
                                    subscribeKey: subscribeKey)
// enable catchup on missed messages
configuration.catchUpOnSubscriptionRestore = true
configuration.stripMobilePayload = false
let pubnub = PubNub.clientWithConfiguration(configuration)  
/**
 * If connection availability check will be done in other way,
 * then use this  function to reconnect to PubNub.
 */
pubnub.subscribe().perform()
