Mobile Push Notifications

Note: Enable Push Notifications from the Admin Dashboard.

PubNub makes it easy to deliver APNS and GCM push notifications. Ideally, clients will be connected to the PubNub network in order to receive published messages in realtime. However, there are situations where this is not always possible (e.g. background processing in iOS), so the Mobile Push Gateway add-on makes it easy to fallback to APNS and/or GCM push notifications. APNS push notifications work with Safari and macOS. GCM push notifications can be configured to work for Chrome.

A common misconception is to mistake subscribing to channels with receiving push notifications for that channel. A user does not need to be subscribed to a channel in order to receive push notifications for it; the only requirement is that their devices have been registered to receive push notifications for the channel. For example, a user subscribed to the channel ‘my_channel’ will not receive push notifications for ‘my_channel’ unless their devices have been explicitly registered to receive push notifications for ‘my_channel’.

Anytime a message is published onto a channel containing a specifically formatted payload (key:value pair) for Apple or Google, PubNub will recognize the published message contains a push notification payload, and will proceed to hand the contents to APNS or GCM along with all of the device_ids registered to receive push notifications on that channel.

Sending push notifications via GCM/FCM

Android devices can use two types of push notifications:

  • Google Cloud Messaging (GCM)/Firebase Cloud Messaging (FCM) is similar to APNS, where PubNub passes the messages off to Google to be delivered.
  • Alternatively, you can use a Background Thread, which allows you to have the same low-latency message delivery. Refer to this knowledge base article on using Android background threads for more details.

First, you must be registered to use Google Cloud Messaging (GCM)/Firebase Cloud Messaging (FCM) for Android.

Each Android device that runs your app has a unique token called its registration ID, which you need to register to be able to send push notifications.

You need to code in either your native app, or use Cordova/PhoneGap with the Cordova Push plugin to obtain the token from your user.

Android device ID size is up to 4096 bytes, and the string starts with APA.

Once you get the device-specific ID, save the token, then use PubNub mobile_gw_provision method and PNmessage object to set GCM/FCM message objects, to finally send a push message to the device.

Sending push notifications via APNS

On Apple devices, you use Apple Push Notification Service (APNS). Essentially, PubNub passes your messages to Apple to be delivered to the user's device(s).

First, you must be registered to use Apple Push Notification Service (APNS).

Each device that runs your app has a unique identifier called a device token, which you need to register to be able to send push notifications.

You need to code in either your native app, or use Cordova/PhoneGap with the Cordova Push plugin to obtain the token from your user.

APNS tokens are about 32 bytes in size.

Once you get the device-specific ID, save the token, then use PubNub mobile_gw_provision method and PNmessage object to set GCM/FCM or APNs message objects, to finally send a push message to the specific device.
